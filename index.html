    Logic Explanation

    This application uses an array to store loot objects. Each object has a name and value property. When the Add Loot button is clicked, validation occurs before pushing the object into the array. A traditional for loop is used to calculate the total loot value and to render the loot list dynamically. Conditional logic is used to validate inputs, check for empty values, prevent negative numbers, and ensure the party size is at least 1 before splitting the loot.
